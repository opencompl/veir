    ParserError and ParserErrorNote #

    These two classes are used to represent structured parse errors with optional source locations. ParserError represents a primary error with an optional source location and a list of secondary notes. Each note is represented by a ParserErrorNote, which has its own message and source location.

                  Parser error formatting #

                  The following functions are used to format a ParserError into a human-readable diagnostic message that includes the source line and a caret pointing to the error location. The main public function is ParserError.format, which takes a ParserError and the source filename and content, and produces a formatted error string.

                  source
                  def Veir.Parser.ParserError.format (e : ParserError) (filename : String) (input : ByteArray) :
                  String

                  Render one primary error label followed by a note label per entry in e.notes, each via formatLabel. The primary location is taken from e.pos.
