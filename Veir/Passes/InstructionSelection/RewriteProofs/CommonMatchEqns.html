    Pre-generated match equations for the RISC-V lowering combinators #

    Tactics like split and grind realize the match_n.eq_k / match_n.congr_eq_k lemmas of a matcher on demand, in whichever module first needs them. The matchers of the lowering combinators live in Veir.Passes.InstructionSelection.RISCV64 and RISCV64Sdag, so two sibling proof modules that both case on the same matcher each bake a private copy of these lemmas (and their _sparseCasesOn_* auxiliaries) into their own olean — and importing both modules together then fails with environment already contains 'Veir.lowerUnaryWLocal.match_3.congr_eq_1._sparseCasesOn_2' (or Veir.lowerBinopNotLocal.match_1.… for the SelectionDAG combinators).

    Generating the lemmas here once, in a module that every rewrite proof imports, gives all of them a single shared copy. The anchor constants pin the modules whose matchers we walk: RISCV64 (the lower*WLocal combinators, via lowerUnaryWLocal), RISCV64Sdag (the SelectionDAG lowerings, via lowerBinopNotLocal), and Interpreter.Basic (the Arith/Llvm/Riscv.interpretOp' matchers the interpretation-unfold proofs case on, via Arith.interpretOp').
